# 力学复习台当前功能总结

本文档基于当前代码实现整理，覆盖前端可用功能、后端处理能力、数据存储方式和现阶段限制。

## 一、产品定位

力学复习台是一个本地优先的理工科期末复习助手，覆盖工科力学类课程，并扩展到数学、物理、化学、生物、电路、信号、控制、热工和计算机基础等课程。核心目标是把课件、PDF、例题文本等资料整理成可复习、可刷题、可追踪的本地资料库，并在不依赖外部服务的情况下提供基础的知识点梳理、复习计划、题目生成和资料问答能力。

系统默认运行在本机，资料和配置保存在项目目录中。只有用户在设置中启用 API 增强版并保存 API Key 后，部分生成类功能才会调用 OpenAI-compatible API。

## 二、科目管理

- 支持新建多个科目，例如材料力学、理论力学、结构力学等。
- 支持切换当前科目，资料、错题、复盘记录都会按科目隔离展示。
- 支持修改科目名称。
- 支持删除科目；删除时会同步移除该科目下的资料记录、错题、复盘记录和本地上传文件。
- 侧边栏会统计当前科目的资料数、错题数和复盘记录数。

## 三、资料导入与管理

### 1. 文件导入

当前支持导入以下资料类型：

- `.pptx`
- `.pdf`
- `.txt`
- `.md`
- 常见图片文件：`.png`、`.jpg`、`.jpeg`、`.webp`

导入后，文件会保存到本地 `data/uploads/` 目录，资料元数据和抽取文本会写入 `data/db.json`。

### 2. 文本抽取

- PPTX：按幻灯片抽取文本，生成“第 N 页”片段。
- PPTX 公式处理：会尽量解析 Office 公式对象，并修正 Symbol 字体中的常见希腊字母和数学符号。
- 公式校验库：内置 89 条参考公式/递推式，覆盖工程力学、大学物理、化学、高等数学、生物学和算法复杂度；公式条目保留变量含义、适用条件、常见误用和开放参考来源。
- PDF：按页抽取可选择文本，生成“第 N 页”片段。
- TXT/MD：直接读取文本，并尝试按例题、题目、章节等结构切分片段。
- 图片：当前只保存原文件，不默认执行本地 OCR，会提示后续可接入图像识别能力。
- 扫描版 PDF 或图片型课件：如果没有可抽取文本，会保留文件并给出解析提示。

### 3. 纯文字例题导入

- 支持在页面中直接粘贴纯文字例题。
- 可填写资料名称。
- 系统会把粘贴内容作为一份文本资料导入，并尝试按“例题”“题目”等标记切分。

### 4. 资料列表与预览

- 资料列表展示文件类型、大小、文本长度、关键词和解析警告。
- 资料列表展示每份资料的“增量知识包”，包括导入后沉淀的知识点、公式、题型、防错点和训练模板数量。
- 支持勾选资料，后续知识地图、复习计划、刷题和问答会优先基于选中的资料。
- 支持全选或取消全选资料。
- 支持预览资料抽取文本。
- 支持查看资料片段目录，例如页码、章节、例题、思考题、练习等。
- PDF 资料支持在页面中嵌入原文件预览，并定位到对应页。
- 支持修改资料名称和抽取文本。
- 支持删除整份资料。
- 支持删除单个资料片段，删除后会重建该资料的抽取文本和关键词。

## 四、知识点与知识地图

系统可基于选中的资料生成知识地图和复习提纲。

### 1. 本地规则生成

本地模式会从资料文本中提取：

- 高频关键词
- 力学主题，例如轴向拉伸与压缩、梁弯曲、扭转、平面应力、压杆稳定、动力学等
- 关键概念
- 公式线索
- 检查点
- 资料证据片段
- 概念关系
- 通用解题流程
- 高频易错点
- 自测问题

### 2. API 增强生成

当设置为 API 增强版且已保存 API Key 时，复习提纲可以由外部兼容 API 生成。知识地图结构仍会保留本地规则结果，便于页面渲染和资料定位。

如果 API 调用失败，系统会自动回退到本地规则生成，并在页面提示原因。

## 五、复习计划

复习计划功能会结合资料章节、专题、错题和已完成复盘记录，生成当前科目的下一步复习安排。

### 1. 计划生成逻辑

系统会优先尝试从资料中识别章节或小节，例如“2-1、轴向拉压的概念和实例”。如果资料结构不明显，则退回到力学主题维度进行规划。

每个计划项包含：

- 复盘主题
- 章节或专题名称
- 资料定位
- 优先级
- 建议时长
- 命中原因
- 相关概念
- 相关公式
- 复盘动作
- 资料证据片段
- 关联错题数量

### 2. 下一步推荐

系统会根据以下因素排序并推荐下一步复盘内容：

- 资料中命中的知识线索数量
- 是否有关联错题
- 未掌握错题数量
- 该主题是否已经复盘过
- 距离上次复盘的时间

### 3. 复盘记录

- 支持对计划项点击“标记完成”。
- 完成后会写入复盘记录，记录主题、章节、时长、关联资料和关联错题。
- 已完成记录会影响下一次计划排序，近期完成的主题会降低优先级。
- 支持删除复盘记录。
- 页面展示最近的复盘历史。

## 六、考前冲刺包

冲刺包面向期末前的短时高效复习，会统计当前科目下选中资料、错题本和复盘记录，生成“今天立刻执行”的压缩清单。

当前能力包括：

- 按资料信号、公式数量、例题/作业、资料易错点、未掌握错题和近期复盘记录计算专题优先级。
- 输出今日执行顺序，包含限时诊断、闭卷复盘、公式适用条件核对、错题回炉和限时刷题。
- 展示统计范围，包括资料数、片段数、文本字数、公式数、未掌握错题数和复盘记录数。
- 给出优先专题、必背公式、易错清单、错题回炉和限时题。
- 支持把冲刺包里的限时题导入刷题区继续答题。
- 支持对冲刺专题标记完成，完成记录会继续影响后续计划和冲刺包排序。

## 七、刷题功能

刷题模块可以基于选中资料生成练习题。

### 1. 题目配置

支持配置：

- 题量：3 到 30 题
- 难度：混合、基础、中等、提高
- 题型：选择、填空、简答、计算

### 2. 本地规则出题

本地模式会根据资料中的关键词、重要句子、公式和力学主题生成题目：

- 选择题：围绕关键词生成概念检查题。
- 填空题：从公式或关键术语中挖空。
- 简答题：要求说明知识点在解题中的作用和适用条件。
- 计算题：根据资料命中的主题套用力学模板，例如轴向拉压、梁弯曲、扭转、平面应力、压杆稳定、单自由度振动等。
- 考试题：从本地考试题型库中补充常见大题入口，例如受力图、轴力图、弯矩图、强度校核、稳定性、影响线、伯努利方程、应力函数、机械振动共振分析、极限积分、牛顿定律、电磁感应、化学平衡、遗传计算、KCL/KVL、电路暂态、卷积、采样、传递函数、劳斯判据、导热热阻、复杂度分析、最短路径和动态规划。
- 教材习题：从本地教材习题风格库中补充分步骤训练题，并给出参考答案、解题步骤、常见错误和评分规则。
- 本地题库覆盖材料力学、理论力学、结构力学、弹性力学、流体力学、机械振动、工程力学综合、高等数学、大学物理、化学、生物学、电路分析、信号与系统、自动控制、热力学与传热学、数据结构与算法。
- 系统会把上传资料中识别出的例题/作业题与本地题库混合排序，优先保留和当前科目、章节、概念匹配的题目。
- 每份资料的增量知识包会补充同类题、公式应用题和易错诊断题；课程级知识包会汇总所有导入资料，作为刷题、考前冲刺包和 API 增强生成的优先上下文。

### 3. API 增强出题

当设置为 API 增强版且 API Key 可用时，题目可以由外部兼容 API 生成。API 失败时会回退到本地规则出题。

### 4. 做题交互

- 支持填写自己的答案或思路。
- 支持查看参考答案和解析。
- 支持删除单道题。
- 支持把题目加入错题本，保存题干、参考答案、解析、用户答案和来源资料。

## 八、资料问答

问答模块支持围绕当前科目和选中资料提问。

- 本地模式会检索相关资料片段，并返回命中的资料、片段标签和摘录。
- API 增强版会把检索到的资料上下文发送给外部兼容 API，由模型组织回答。
- 如果 API 不可用，会回退到本地检索回答。
- 没有相关片段时，会提示更换关键词或接入 API。

## 九、错题本

错题本用于沉淀刷题中的错误和考前需要重看的题。

当前能力包括：

- 从刷题结果一键加入错题。
- 保存用户答案、参考答案、解析和来源资料。
- 标记错题是否已掌握。
- 删除错题。
- 基于错题生成同类题。
- 错题会参与复习计划排序，未掌握错题会提高相关主题优先级。
- 删除资料时，会自动移除错题中对该资料的引用。
- 删除错题时，会同步移除复盘记录中对该错题的引用。

## 十、设置与生成模式

设置页支持两种生成模式：

- 本地规则版：默认模式，不调用外部服务。
- API 增强版：使用 OpenAI-compatible Chat Completions API 增强总结、出题、问答和同类题生成。

可配置内容包括：

- API Base URL
- 模型名
- API Key
- 清除已保存 API Key

API Key 会写入本项目的 `data/db.json`。前端读取状态时不会返回真实 Key，只会返回已设置标记。

## 十一、后端与接口能力

后端使用 Node.js 原生 HTTP 服务实现，主要能力包括：

- 静态资源服务：提供 `public/index.html`、`public/app.js`、`public/styles.css` 和运行时依赖的 `lucide`、`marked`。
- 本地数据库读写：使用 `data/db.json` 保存课程、资料、错题、复盘记录和设置。
- 上传文件保存：使用 `data/uploads/` 保存原始上传文件。
- 资料解析：支持 PPTX、PDF、TXT、MD 和图片占位处理。
- 本地知识图谱、复习计划、题目、问答和同类题生成。
- API 增强调用与失败回退。

主要 API 包括：

- `GET /api/state`：读取当前公开状态。
- `POST /api/courses`：新建科目。
- `PATCH /api/courses/:id`：修改科目。
- `DELETE /api/courses/:id`：删除科目。
- `POST /api/upload`：上传并解析资料。
- `POST /api/text-examples`：导入纯文字例题。
- `PATCH /api/documents/:id`：修改资料名称或文本。
- `DELETE /api/documents/:id`：删除资料。
- `DELETE /api/documents/:id/units/:index`：删除资料片段。
- `POST /api/generate/summary`：生成知识地图和复习提纲。
- `POST /api/generate/plan`：生成复习计划。
- `POST /api/generate/cram-pack`：生成考前冲刺包。
- `POST /api/generate/quiz`：生成题目。
- `POST /api/generate/similar`：基于错题生成同类题。
- `POST /api/ask`：资料问答。
- `POST /api/mistakes`：新增错题。
- `PATCH /api/mistakes/:id`：更新错题掌握状态或用户答案。
- `DELETE /api/mistakes/:id`：删除错题。
- `POST /api/sessions`：新增复盘记录。
- `DELETE /api/sessions/:id`：删除复盘记录。
- `POST /api/settings`：保存生成模式和 API 配置。

## 十二、测试能力

项目提供两个脚本：

- `npm run test:core`：核心逻辑测试，不需要启动网页服务。覆盖 PPTX/PDF 文本抽取、Symbol 字体和公式解析、关键词提取、知识地图、复习提纲、复习计划、考前冲刺包、刷题和同类题生成。
- `npm run smoke`：接口烟测，需要先启动服务。检查首页和 `/api/state` 是否可访问。

## 十三、启动方式

常规启动：

```bash
npm start
```

默认访问地址：

```text
http://127.0.0.1:4173
```

如果当前环境没有可用的 `npm`，可以直接使用项目 README 中给出的 Node 运行时路径启动 `server.cjs`。

## 十四、当前限制

- 图片和扫描版 PDF 当前不做本地 OCR，只保存原文件并提示解析限制。
- PPTX/PDF 文本抽取依赖文件自身是否包含可读文本；纯图片课件无法直接生成高质量知识点。
- 本地规则生成适合基础复习整理，但语义理解能力有限；复杂总结、开放问答和更高质量出题建议启用 API 增强版。
- API Key 存在本地 `data/db.json` 中，没有额外加密机制。
- 复习计划目前基于规则排序，不包含真正的间隔重复算法或日历排程。
- 数据存储为单个 JSON 文件，适合个人本地使用，不适合多人并发或大规模资料库。
